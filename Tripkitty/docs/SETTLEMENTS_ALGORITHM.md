# Алгоритм расчёта взаиморасчётов

Алгоритм решает задачу: **как погасить все долги внутри группы с минимальным количеством переводов**.

---

## Шаг 1 — Баланс каждого участника

Для каждого расхода:
- Плательщик получает **+сумма** (он вложил деньги)
- Каждый участник получает **−своя доля** (он потребил)

Поддерживается три типа разбивки:

| Тип | Логика |
|---|---|
| **Equal** | `сумма / количество участников` |
| **ByShares** | `сумма × вес участника / сумма всех весов` |
| **ByAmounts** | конкретная сумма для каждого участника |

После обхода всех расходов у каждого участника есть итоговый баланс:
- **> 0** — заплатил за других, ему должны вернуть (кредитор)
- **< 0** — за него платили другие, он должен (должник)
- **= 0** — всё сошлось

---

## Шаг 2 — Минимизация транзакций

Алгоритм жадно гасит долги:

1. Отсортировать кредиторов по убыванию баланса
2. Отсортировать должников по возрастанию (самый большой долг первым)
3. Взять первого кредитора и первого должника, погасить их долги друг об друга
4. Кто из них обнулился — взять следующего из своей группы
5. Повторять до конца

---

## Пример

### Расходы поездки

| Расход | Сумма | Платил | Участники |
|---|---|---|---|
| Такси | 863₽ | Дмитрий | Дмитрий, Юля К., Юля С. |
| Такси | 345₽ | Дмитрий | Дмитрий, Юля К., Юля С. |
| Чаевые | 600₽ | Никита | Все 6 |
| Кальян | 2 100₽ | Никита | Даша |
| Бла бла кар | 3 600₽ | Юля С. | Даша, Юля К., Юля С., Дмитрий |
| Ужин | 6 850₽ | Никита | По суммам (Дмитрий 450, Юля С. 1150, Даша 1450, Г.Д. 1450, Никита 1300, Юля К. 1050) |

### Балансы участников

| Участник | Заплатил | Должен был | Баланс |
|---|---|---|---|
| Никита | 9 550₽ | 1 400₽ | **+8 150₽** |
| Юля С. | 3 600₽ | 2 552.67₽ | **+1 047.33₽** |
| Дмитрий | 1 208₽ | 1 852.67₽ | **−644.67₽** |
| Г.Д. | 0₽ | 1 550₽ | **−1 550₽** |
| Юля К. | 0₽ | 2 452.67₽ | **−2 452.67₽** |
| Даша | 0₽ | 4 550₽ | **−4 550₽** |

### Работа алгоритма

```
Кредиторы (кому должны):  Никита +8150,  Юля С. +1047.33
Должники  (кто должен):   Даша −4550,    Юля К. −2452.67,  Г.Д. −1550,  Дмитрий −644.67

Итерация 1: Никита (8150) ↔ Даша (−4550)
  → Даша → Никита 4550₽
  Никита остаток: 3600

Итерация 2: Никита (3600) ↔ Юля К. (−2452.67)
  → Юля К. → Никита 2452.67₽
  Никита остаток: 1147.33

Итерация 3: Никита (1147.33) ↔ Г.Д. (−1550)
  → Г.Д. → Никита 1147.33₽
  Никита = 0 ✓   Г.Д. остаток: −402.67

Итерация 4: Юля С. (1047.33) ↔ Г.Д. (−402.67)
  → Г.Д. → Юля С. 402.67₽
  Г.Д. = 0 ✓   Юля С. остаток: 644.66

Итерация 5: Юля С. (644.66) ↔ Дмитрий (−644.67)
  → Дмитрий → Юля С. 644.66₽
  Все балансы = 0 ✓
```

### Результат

| От | Кому | Сумма |
|---|---|---|
| Даша | Никита | 4 550₽ |
| Юля К. | Никита | 2 452.67₽ |
| Г.Д. | Никита | 1 147.33₽ |
| Г.Д. | Юля С. | 402.67₽ |
| Дмитрий | Юля С. | 644.66₽ |

5 переводов вместо потенциально большего числа прямых долгов между участниками.

---

## Почему результат может казаться неинтуитивным

Алгоритм не сохраняет связь «кто у кого конкретно занял». Он ищет математически оптимальный путь.

Например, Дмитрий физически участвовал в расходах где платил Никита (чаевые, ужин), но в итоге платит Юле С. Это потому что баланс Никиты был полностью закрыт другими должниками раньше, чем очередь дошла до Дмитрия.

Аналогия: все скидываются в общий котёл, котёл раздаёт деньги тем кто переплатил. Алгоритм симулирует этот котёл без реального посредника.

---

## Округление

При делении поровну возникает погрешность в копейки:

```
863₽ ÷ 3 = 287.666...₽ → 287.67₽
287.67₽ × 3 = 863.01₽ ≠ 863₽
```

Из-за этого сумма всех долгов может отличаться от суммы всех кредитов на 0.01–0.02₽. Это нормальное поведение — аналогично работают Splitwise и Tricount.
